Log #2: Configuring GPIO & Interrupts

Resolved
📅 Date: June 4, 2026
The Goal

Configure an onboard switch (e.g., Switch S1 mapped to P1.2) to toggle an LED using interrupts rather than polling.

The Obstacle / Bug

The interrupt routine wasn't triggering when the button was pressed. The microcontroller either ignored the press entirely or entered an infinite loop state.

How I Resolved It
  • Internal Pull-up: Realized the switch pin needed the internal pull-up resistor explicitly enabled because the hardware design defaults to active-low.
  • Unlock I/O: Remembered to disable the GPIO high-impedance mode by clearing the LOCKLPM5 bit in the PM5CTL0 register (crucial for MSP430 FRAM devices!).
Code Snippet
// Unlock GPIO high-impedance mode
PM5CTL0 &= ~LOCKLPM5;

// Configure P1.2 as input with pull-up resistor
P1DIR &= ~BIT2; // Set P1.2 as input
P1REN |= BIT2;  // Enable resistor
P1OUT |= BIT2;  // Select pull-up

// Configure Interrupt
P1IES |= BIT2;  // High-to-low transition
P1IE  |= BIT2;  // Enable P1.2 interrupt
__enable_interrupt(); // Enable global interrupts

Log #1: Setting Up Code Composer Studio & Blinking LED

Resolved
📅 Date: June 1, 2026
The Goal

Establish the development environment using TI Code Composer Studio (CCS) and flash a basic Blinky program to verify the board functions.

The Obstacle

CCS threw a driver communication error when attempting to flash the firmware, failing to recognize the MSP-FET onboard debugger.

How I Resolved It

Updated the TI emulators pack via the CCS App Center and switched to a high-quality USB data cable. Verified target configuration was strictly set to MSP430FR4133.
